Overriding Equals method - @override equals().

July, 2018

Sometimes we may need to know if two objects of a value class contains same value, predominantly this could be the major reason for a developer to override equals method of Object class; if anything else could be reason better not override equals because in anyother case an object of a class will always be equal only to itself.

However, having liberty of overriding the equals method, we should know when not to, how to and also when to override equals method.

When not to ?

  • If instance of your class is inherently unique. for example instance of Thread class, the implementation of equals in Object class itself is suffice for Thread to perform at fullest.
  • When you dont require to check ("logical equality test") if the values of the object are same.
  • When parent class has its equals overidden.
  • If your know that equals of the class never gonna be called.
  • If your class is instance controlled - eg: singleton.

How to? You will have to adhere some general contracts while you override equals.
The explanations and examples assume the object reference are always not-null objects and values

Reflexive:

For any value x in an object, x.equals(x) should be true. 
                                        
                                            public class Example{

                                                private String x;
                                                
                                                public String getX(){
                                                    return this.x;
                                                }
                                                
                                                public void setX(){
                                                    this.x = x;
                                                }
                                                
                                                @override public boolean equals(Object o){
                                                
                                                    if( !o instanceof Example){
                                                        return false;
                                                    }
                                                    if( this.x.equals((Example)o)){
                                                    }
                                                    return false;	
                                                }
                                            }

                                            public class Main{

                                                public static void main(String args[]){
                                                    List<Example> exampleList = new ArrayList();
                                                    Example example = new Example();
                                                    exampleList.add(example);		
                                                    System.out.println(exampleList.contains(example));	
                                                }
                                            }

output: false

Any inappropriate equal override will return false for a simple check of an object existence in a Collection after adding into it.

Symmetric:

For any value of x in an object, if x.equals(y) is true then y.equals(x) also should be true.

First let us understand how this could fail, 

                                        
                                            public class Example{

                                                private String x;
                                                
                                                public Exmaple(String x){
                                                    this.x = x;
                                                }
                                                
                                                public String getX(){
                                                    return this.x;
                                                }
                                                
                                                public void setX(){
                                                    this.x = x;
                                                }
                                                
                                                @override public boolean equals(Object o){
                                                    if( o instanceof Example){
                                                        return	x.equalsIgnoreCase(((Example)o).x);
                                                    }
                                                    if( o instanceof String ){
                                                        return	x.equalsIgnoreCase((String)	o); 
                                                    }
                                                }
                                            }

What happens ? Say we have,

Example ex = new Example("great");
String st = "great";

ex.equals(st) returns true
Ok, but
st.equals(ex) retuns false
beacause st a String type has no possibility of knowing what cis is and failed. A perfect symmetrical failure.

What should have been done ?

The basic mistake here is we are trying to interoperate two different types, we will have to eliminate this inappropriate comparison by refactoring the equals overiride into 

                                        
                                            @Override public boolean equals(Object o) { 
                                                return o instanceof Example && 
                                                    ((Example) o).x.equalsIgnoreCase(x); 
                                            }

This will return false when two different types are compared.
